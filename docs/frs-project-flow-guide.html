Our mission & motto

Core motto: Field → Approve → Bill — capture trustworthy quantities in the field, lock them through manager approval, and export clean billing backup for pay applications.

Pavement Marking

Lane lines that keep America moving

Capture daily LF, STA ranges, and tickets from the stripe truck — ready for pay-app backup.

Traffic Control

Work zones, certified and accountable

Log setups, certifications, and crew notes so office billing never chases missing docs.

Permanent Signs

Every post, every station, verified

Point-location quantities and photos tied to bid items for clean progress billing.

Daily field cycle

  1. Field Lead opens assigned project → picks report date & crew size
  2. Enters quantities per bid line (STA range, manual LF, or single location)
  3. Attaches photos / tickets / certifications
  4. Saves draft or submits — report locks on submit
  5. Manager reviews pending queue → approves or returns with mandatory comment
  6. Project Admin opens Billing when job is billing ready (approved + no pending)
  7. Exports CSV backup for pay application

Admin setup order (before first field report)

  1. Units

    LF, EA, LS, etc. — used on bid master dropdown

  2. Project types

    Pavement marking, traffic control, signs, combined scopes

  3. Bid master

    Master bids + sub-bids by division; CSV import supported

  4. Users

    Field leads, managers, project admins — assign roles

  5. Project

    Job number, name, divisions, PM, route map, bid lines

  6. Project tasks

    Line code, conversion factor, assigned field person per task

Report lifecycle

Statuses

Draft → Submitted (locked) → Returned | Approved | Approved with notes

Business rules

  • Return comment is mandatory
  • Reported LF = Physical LF × Conversion Factor
  • Billing export only when zero pending approvals
